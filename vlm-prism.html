사용자 매뉴얼
앱 조작 절차와 VLM·MODAPTS 기반 동작분석 산출 방식을 확인합니다.
기본 사용 흐름
1. 영상 업로드
업로드 & 분석 탭에서 제조 공정 영상을 선택합니다. 영상이 로드되면 해상도, 길이, 예상 분석 프레임 수가 표시됩니다.
2. 공정 조건 입력
공정명, 작업자 ID, 표준시간, 분석 모드, 샘플링 FPS, 부품 수를 입력합니다. 표준시간은 분석 효율 계산의 기준이 됩니다.
3. 분석 실행
분석 시작 버튼을 누르면 프레임 추출, 손/부품 추적, VLM 분석, MODAPTS 코드 생성, 결과 계산 순서로 진행됩니다.
4. 결과 확인
분석 결과 탭에서 KPI, 타임라인, 코드별 분포, 동작 유형 비율, 낭비 동작, 상세 동작 목록을 확인합니다.
편집 및 출력
MODAPTS 코드 편집
상단 역할을 엔지니어 또는 PE/산업공학자로 변경한 뒤 MODAPTS 편집 탭을 엽니다. 분석으로 생성된 각 동작 행에서 코드 드롭다운을 선택하면 MOD 수와 예상 시간이 즉시 갱신됩니다.
편집 행 조작
동작 설명 입력칸에서 작업 내용을 보정할 수 있습니다. 누락된 동작은 행 추가 버튼으로 새 행을 만들고, 불필요한 동작은 삭제 버튼으로 제거합니다. 하단 수정 후 합계에서 총 동작 수, 총 MOD, 예상 시간, 수정 후 효율을 확인합니다.
저장 및 재계산
저장 및 재계산을 누르면 현재 편집 테이블 순서대로 시작·종료 시간이 다시 배정되고, 수정된 코드 기준으로 MOD 합계, 예상 시간, 효율, 타임라인, 보고서가 갱신됩니다.
보고서 내보내기
보고서 & 내보내기 탭에서 CSV, JSON으로 결과를 저장하거나 화면에 보이는 보고서를 PDF로 다운로드합니다.
운영자 역할에서는 편집 메뉴가 비활성화됩니다. 분석 검토와 결과 출력은 가능하지만 코드 보정은 엔지니어/PE 역할에서 수행하도록 설계되어 있습니다.
동작분석 산출 흐름
영상 입력작업 영상 업로드 및 기본 메타데이터 확인
프레임 샘플링설정 FPS 기준으로 분석 대상 프레임 추출
동작 인식손, 부품, 공구의 이동과 상호작용 추정
코드 매핑동작을 M, G, P 등 MODAPTS 코드로 변환
시간 산출MOD 수와 0.129초 계수로 표준 시간 계산
VLM 개요
VLM의 역할
VLM은 영상 프레임의 시각 정보를 해석해 작업자의 손 이동, 부품 접근, 쥐기, 놓기, 정렬 같은 의미 단위 동작을 추정합니다.
보조 추적 정보
OpenCV와 Mediapipe 기반의 손 위치, 이동 방향, 거리 변화 같은 신호를 함께 사용해 동작 구간과 후보 동작을 안정화합니다.
시뮬레이션 모드
현재 웹앱은 실제 서버 분석 대신 브라우저에서 샘플 동작 패턴을 생성해 VLM 분석 결과처럼 시각화하는 데모 구조입니다.
MODAPTS 산출 방식
코드 체계
M은 이동, G는 쥐기, P는 놓기, R은 재파지, A는 압력, E는 시선, F는 손가락 동작을 나타냅니다. 각 코드는 고유 MOD 값을 가집니다.
시간 계산
각 동작 시간은 MOD 수 × 0.129초로 계산합니다. 예를 들어 M3는 3 MOD이므로 0.387초로 산출됩니다.
효율 계산
표준시간은 입력된 표준 MOD × 0.129초이며, 효율은 표준시간을 분석 총시간으로 나누어 백분율로 표시합니다.
낭비 동작 판정
MOD 값이 큰 동작, 복잡 파지(G3), 압력 동작(A14) 등은 개선 검토 대상으로 표시되어 레이아웃, 지그, 공구 개선 힌트를 제공합니다.
